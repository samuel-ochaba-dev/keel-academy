# Depth profiles

Use this reference to match the answer length and density to the user’s actual ask.

## Profile 1: Quick definition

Use when the user asks:

- “What is X?”
- “Explain X simply.”
- “Quickly tell me what this means.”

### Shape

- plain-English explanation
- one analogy or example
- one sentence of nuance if needed

### Goal

The user should walk away with a usable mental handle.

## Profile 2: Working understanding

Use when the user asks:

- “Help me understand X.”
- “What’s the difference between X and Y?”
- “How does X work?”

### Shape

- clean explanation of the core idea
- mechanism in simple steps
- contrast with a nearby concept
- one concrete example
- one misunderstanding to watch for

### Goal

The user should be able to use the concept correctly.

## Profile 3: Deep understanding

Use when the user asks:

- “Help me deeply understand X.”
- “Teach me X from first principles.”
- “Explain this well enough that I can teach it.”

### Shape

- start simple
- define the key parts
- explain how the parts interact
- layer in nuance gradually
- compare neighboring concepts
- include edge cases, limits, or debates
- end with a short self-test

### Goal

The user should be able to explain the idea back clearly and avoid the common traps.

## Escalation rule

If you are unsure which profile to use, start lower.

It is better to be briefly too simple and then deepen on request than to overwhelm the user in the first reply.
