# Explanation flow

Use this reference when the concept needs a more deliberate teaching shape.

The answer should still **read naturally**. The reader should feel guided, not marched through headings.

## Invisible flow

A good explanation usually moves through this sequence:

1. **Open with relevance**  
   Start near the user’s world. Show where the idea shows up or why it matters in practice.

2. **State the clean version early**  
   Say the core idea in the simplest accurate sentence you can.

3. **Anchor with familiarity**  
   Use an analogy, comparison, or familiar system to give the user something to hold onto.

4. **Make it concrete**  
   Use a specific example. Prefer real or realistic examples over vague placeholders.

5. **Layer in nuance**  
   Add one new moving part at a time. Connect each new layer back to what was already established.

6. **Leave the user with a check**  
   End with a quick question, misconception, or recognition signal so they can test whether it clicked.

## Compression rule

If the topic is simple, compress the flow.

If the topic is deep, keep the same flow but spend more time on the middle layers.

## Transition language

Useful phrases for keeping the explanation smooth without visible section headers:

- “The simplest way to think about it is…”
- “A good mental model is…”
- “Here’s where that becomes real…”
- “Now the subtle part is…”
- “What people often mix up is…”
- “You know you’ve got it when…”

## What to avoid

- announcing the teaching framework out loud
- dumping all nuance before the core idea is stable
- switching metaphors midstream without warning
- using examples that are memorable but not representative
